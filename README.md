## 📌 process

### 1. 기획 & 와이어프레임 생성

- 프로젝트 목표 & 핵심 기능 정의

- 기능 별로 task 분류

### 2. 깃허브 이슈 등록

- task별로 묶어서 이슈 생성

- label, milestone 설정

### 3. 브랜치 전략

- main(운영), dev(테스트), feature(기능별 개발)로 구성

- task별로 feature 브랜치 생성 (ex. #1-main-page)

### 4. 개발 & pr, merge 관리

- 작업 단위를 나눠서 커밋하고 이슈 번호 포함

- 반드시 pr 생성 후 셀프 코드리뷰 진행

- 리팩토링 후 merge 하며 close, 브랜치 삭제

### 5. 배포 & 태그 관리

- ci/cd 연결

- release tag 추가 후 배포
